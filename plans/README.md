## Findings considered and rejected (do not re-audit)

- **vmstatus notifier "race/panic on closed channel"** (3 variants): `Subscribe`'s cleanup closes the channel under `mu.Lock()`; `broadcast` sends under `mu.RLock()`. Writer and readers are mutually exclusive — the close cannot interleave with a send. Correct as written (`internal/proxmox/vmstatus/notifier.go`).
- **Mirror reconcile goroutine leak**: `running` flag under mutex guarantees a single goroutine; `pending` coalesces bursts (`internal/proxmox/mirror.go:41-77`).
- **SSE cleanup race** (`handlers/events.go`): receiving from a closed channel is safe in Go; the loop nils the channel on `!ok`. No race.
- **Inventory SSE permission leak**: events are stripped to type+timestamp (`event.ItemID = nil`, events.go:141); VM statuses are per-principal filtered via `Authz.FilterVisibleStatuses`. By design.
- **VNC done-channel "hang"** (vnc.go:213-222): unblocks on the next failed write after either side closes; acceptable for a VNC bridge. Not worth restructuring.
- **Missing TanStack Query `staleTime`**: a global 30s default exists (`apps/web/src/router.tsx:9`).
- **SameSite=Lax cookies / missing CSRF tokens**: Lax is the correct default for this single-origin SPA; refresh rotation with family revocation (`internal/auth/sessions.go`) is well implemented.
- **Committed secrets**: no `.env*` tracked or in git history (verified via `git ls-files` and `git log --all -- '*.env*'`). Local-only by documented design. (Rotation recommendation lives in plan 005.)
- **`PROXMOX_INSECURE` / `LDAP_INSECURE`**: opt-in flags defaulting to false; a deployment-configuration concern, not a code defect.
- **golang.org/x/crypto|net|sys "lagging security patches"**: the CVEs cited by the auditor are fixed in the pinned versions; advisory `govulncheck` added to CI in plan 001 instead.
- **SDN feature "half-built"**: fully built — route (`apps/web/src/routes/_dashboard/admin/sdn.tsx`), page, dialog, columns all exist.
- **Refresh tokens not bound to UA/IP**: fields stored but unchecked; deliberate usability trade-off, low value for an internal tool. Revisit only if threat model changes.
- **750ms clone/publish progress polling**: bounded to active operations; SSE migration is M effort for marginal gain. Not now.
- **Unbounded list pagination / table virtualization**: no evidence of scale problems at ~100 users; revisit with real data.
- **TanStack version pinning**: `bun.lock` is committed; ranges don't drift in practice.
- **mongo-driver indirect dep**: transitive, not imported, not linked into the binary. Noise.
- **LDAP bind password memory-zeroing**: impractical in Go (string copies), negligible risk reduction. Rejected.
- **parseIntParam error-flow "bug"**: callers check the ok/err return; no defect found.
- **Stale `itemId` closure in `vnc-console.tsx` / EventSource unmount handlers**: not verified by the advisor; LOW priority. If symptoms appear (console connects to wrong VM after fast navigation), investigate then.
